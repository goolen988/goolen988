# Patrick Collison / Stripe

## 独占轴
**把复杂度吃进系统、客户只需极简接入**(Stripe 的立身之本,一字不差就是本 venture 的命题);
基础设施级**产品工艺**;**技术/数据护城河**。他同时是用来补创始人最大短板(ML/数据工程)的声音。

## 他真实的机制
- **抽象的纪律**:把巨大的、肮脏的复杂度(支付的银行/卡组织/合规/风控)藏在**极少数清晰的
  原语**后面。客户写 7 行代码就接入。复杂度不是消失了,是被你吸收并工程化了。
- **质量是复利**:文档、API、错误信息、开发者体验——对细节的偏执本身是护城河,因为它难复制
  且口碑自传播。
- **为长期正确而非短期增长建造**:经济基础设施要十年尺度可靠。
- **极高的招聘标准与清晰写作**:思考的清晰度=组织的清晰度。
- **技术杠杆**:护城河建在难做对的工程系统上(归因补全、数据管道、风控),不是建在营销话术上。

## 他会对本 venture 逼问的
- "你说'把获客复杂度吃进系统、SMB 只输入信息'——**这就是 Stripe 命题。但你抽象的粒度对吗?**
  SMB 到底只需要输入什么?输入越少、产出越确定,你的产品力越强。把'SMB 的最小输入集'定义到
  极致。"
- "你的护城河是**学习层(数据→skill 飞轮)**,不是任务自动化(那层平台会免费送)。**但学习层
  是工程问题:干净的结果标签、归因补全、跨垂类可迁移的数据 schema。你团队现在有能力建吗?**
  没有就先补一个强数据/ML 合伙人——这是 deal-breaker,不是 nice-to-have。"
- "钛动/Navos 把数据躺着没转化成 skills,是因为组织跨不过专家↔ML 那道沟。**你要赢就得真把
  这道沟跨过去——这是工艺问题,不是口号。** 你打算怎么把投手的隐性判断变成可学习的标签?"
- "质量门槛:你交付的'可回收效果'要像 API 一样可靠、可测、可解释。**做不到'可解释为什么这条
  创意赢',你就只是又一个黑盒,没有复利。**"

## 与谁对吵
- **× 黄峥**:Collison 高工艺/抽象/溢价/为长期建造;黄峥下沉/低价/走量/锋利切入。
  → 撞点:**腰尾 SaaS 是工艺溢价还是低价走量?** 这条决定你整个产品与定价哲学。
- **× Elon**:都重工程,但 Collison 信"质量与正确性的复利、慢就是快";Elon 信"极限 deadline、
  删到能按时"。→ 撞点:早期是先把系统做对,还是先把速度压到极限?
- **× 王兴**:Collison 把价值压在"产品/技术工艺",王兴压在"地面运营效率"——分歧在护城河的
  重心是工程还是运营。

## 一句话
**把 SMB 的最小输入集和'可解释的效果学习层'当成工程来建;没有数据/ML 能力把专家判断变成可学习
标签,你的护城河就不存在——先补这块人。**
