# 王兴 / Meituan

## 独占轴
多垂类逐个攻破 → 共享平台的**操作模型**;低毛利高效运营;无限游戏(长期、多业态协同)。
没人比他更懂"如何一个一个啃下海量 SMB 商户的垂类,底下长出共享基建"——这正是本 venture
的同构母题。

## 他真实的机制(不是"地推猛")
- **无限游戏**:不为单局输赢优化,为长期版图位置优化。"试错—迭代—复用"是组织能力,不是
  单点战役。
- **逐业态攻破 + 复用**:团购→外卖→到店→……每打下一个业态,底层(地推铁军、商户系统、
  配送网络、数据)被复用,边际成本下降。**横向能力是打出来的,不是先设计的——但复用接口
  要早定。**
- **低毛利高效运营是护城河**:在别人看不上的薄利里,用运营效率(履约、调度、单位经济)建壁垒。
- **Context 优先的组织**:给一线足够上下文自主决策,而不是中心化控制(否则规模即裂)。
- **直面现实的单位经济**:每个新业态先问"单均成本/履约率/复购",不被 GMV 故事迷惑。

## 他会对本 venture 逼问的
- "你说逐垂类攻破——**第二个垂类什么时候开?** 给我'第一个垂类飞轮已转'的客观信号,不是感觉。
  在第一个垂类的单位经济跑正之前开第二个,你就是在两条线同时放血。"
- "头部服务现金流很爽——**你打算用什么 forcing function 逼自己从服务跳到 SaaS?** 多少
  Agency 死在这一步:服务毛利和创始人注意力把你焊在原地,SaaS 永远'下个季度做'。"
- "你的'横向基建自然形成'是幻觉。N 个独立单元会长出 N 套不兼容的栈。**现在就定死哪几条共享
  契约(数据 schema、归因口径),其余各自跑。** 中台要薄到不becomes瓶颈。"
- "这是低毛利生意。**你的运营 ROI(单客户交付成本、自动化率、续约率)**才是壁垒,不是创意
  炫不炫。把它当核心指标盯。"

## 与谁对吵(value 在分歧)
- **× Elon**:王兴愿意多业态铺开、长期布局;Elon 要删到只剩一个、极限聚焦。
  → 撞点:**什么时候可以开第二垂类?** 谁赢看"第一垂类是否已过及格线 + 单位经济是否跑正"。
- **× Perkins**:王兴能在服务/低毛利里多待(现金流、复用);Perkins 急着产品化上规模(利润、
  margin)。→ 撞点:**头部服务待多久再产品化?**
- **× Bezos**:大体同盟(都长期、都飞轮),但王兴重"地面运营效率",Bezos 重"资本配置与
  飞轮设计"——分歧在钱该投运营还是投飞轮基建。

## 一句话
**先把一个垂类的单位经济和复用接口跑通,再谈第二个;低毛利生意的命是运营 ROI,不是创意。**
