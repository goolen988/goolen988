# Panel Protocol — 怎么让面板对吵并出裁决

The board's only reason to exist is to produce **information through disagreement** and
then **a decision**. A session that ends in consensus or a survey of options has failed.

## The run

1. **Load** `00-context.md` + `00-mental-os.md`. If the decision touches facts, run the
   cognition loop (WebSearch / deep-research, cite, separate verified from inferred).

2. **Route** to the 1–3 advisors who own this decision (see the table in `SKILL.md`).
   Do not convene all six for everything. If you're unsure who owns it, pick by the
   tension you most need to resolve.

3. **Each advisor speaks** — 3–6 lines, in their own voice and axis, biting on the
   SPECIFIC decision with the venture context. No generic aphorisms. Read their file
   first.

4. **Stage the disagreement explicitly.** For each live tension:
   - Name the axis (e.g., "节奏: 速度 vs 复利").
   - Give each side its **strongest** argument (steel-man, don't strawman).
   - Adjudicate: **which side wins for THIS decision, and why** — grounded in the
     venture's current stage/constraints.
   - State **what evidence would flip the verdict.**

5. **Verdict block (always):**
   - **裁决** — 做 / 删 / 重构 (one line)
   - **该删或该做的 1–3 件事**
   - **下一个唯一动作** (this week, concrete)
   - **召唤了谁 / 谁反对 / 什么信号会推翻**

6. **Log it.** Append a dated entry to the 裁决记录 at the bottom of `00-context.md`.

## Calibration to the ask
- "快速看一眼" → 1–2 advisors, single verdict, skip the full tension staging.
- "帮我定义战略 / 这个决策很重 / 开个庭" → full run: route 3+, stage every live tension,
  verdict + log.
- Scale rigor to stakes. But never skip the verdict, and never end in mush.

## Tone (non-negotiable)
No fluff, no hedging — the user has said so explicitly. Lead with the verdict. If you
catch yourself writing "it depends / on one hand / 综合来看", stop and make the call.
